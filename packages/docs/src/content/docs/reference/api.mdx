## REST Endpoints

The runtime auto-generates REST endpoints for every entity defined in your blueprint:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/:entity` | List records |
| `GET` | `/api/:entity/:id` | Get single record |
| `POST` | `/api/:entity` | Create record |
| `PUT` | `/api/:entity/:id` | Update record |
| `DELETE` | `/api/:entity/:id` | Delete record |

All endpoints respect entity-level access control rules defined in your blueprint. Requests can authenticate via session cookies or API keys.

## Query Parameters

List endpoints support filtering, sorting, and pagination:

```
GET /api/Post?status=published&orderBy=createdAt&order=desc&limit=10
```

| Parameter | Description |
|-----------|-------------|
| `orderBy` | Field to sort by |
| `order` | Sort direction (`asc` or `desc`) |
| `limit` | Maximum number of results |
| Any field name | Filter by field value (e.g., `status=published`) |

## Authentication

### Session-Based

Browser requests use session cookies automatically. Pages with `auth = "required"` redirect to the login page.

### API Keys

For programmatic access, send an API key in the `Authorization` header:

```
Authorization: Bearer <api-key-value>
```

API keys are defined in the blueprint:

```toml
[[auth.apiKeys]]
name = "my-agent"
keyEnv = "MY_API_KEY"
```

API key requests skip CSRF validation but still go through access control.

## OpenAPI

Zebric generates an OpenAPI specification from your blueprint. Access it at:

```
GET /api/openapi.json
```

The spec includes all entities, their fields, and available operations. Use it with:

- API clients (Postman, Insomnia)
- Documentation generators (Swagger UI, Redoc)
- AI agents and code generators
- Type generation tools
